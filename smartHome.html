HCI Smart Home User Interface

Purpose

In my Human Computer Interaction Class, we were tasked with building a web or mobile based application. I served as the UI designer for my group. We collectively decided to challenge ourselves to create a prototype for a smart home control system as it this concept becoming ubiquitous.

Design Goal

Create an application that allows a user to control smart aspects of their home from anywhere.

User Needs

  • to have a personalized experience
  • to operate the application from anywhere with internet service
  • to be able to operate the device independently, indefinitely
  • to add and remove IOT smart home devices

    • Testing Process

  • Usability Testing (first round)
  • Heuristic Review
  • Usability Testing (second round)

    • Design Process

    Starting with a paper prototype, we had an iterative design process. Each member of the team conducted a cognitive walkthrough and we compared notes to alter the current design in the first mockups. Once we had the screens, we used InVision to build the interactivity of our prototype. I then conducted a heuristic review using Neilsen's Design Heuristics. I found that we needed to focus on user control and consistency between pages, as well as matching the design to users' mental models.
